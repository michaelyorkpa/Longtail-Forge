Longtail Forge uses modules for major workflow areas and optional features. First-party modules ship with Longtail Forge, such as Time Tracking, Tasks, Notes, Lists, Tags, and Users. Framework features are shared app services such as workspaces, permissions, search, files, settings, notifications, and Help.

Enabled modules can contribute navigation, settings, permissions, APIs, search types, notifications, files, tags, Workbench context, and Help content. A disabled module is hidden from normal navigation and active search targets. Historical or recovery behavior can still show existing records where the framework allows it.

Third-party modules use the same ownership model when they are supported by the installed app: they own their workflow behavior and module Help, while shared framework services keep the app shell, permissions, files, search, tags, notifications, and Help contracts consistent.

Module-authored Help articles appear only when their module is active and visible in the workspace. Framework Help articles stay available across workspaces.
