Longtail Forge uses first-party modules for major workflow areas such as Clients and Projects, Time Tracking, Tasks, Tags, and Users. Enabled modules contribute navigation, settings, permissions, APIs, search types, notifications, and Help content.

A disabled module is hidden from normal navigation and active search targets. Historical or recovery behavior may still show existing records where the framework allows it.

Module-authored Help articles appear only when their module is active and visible in the workspace. Framework Help articles stay available across workspaces.
