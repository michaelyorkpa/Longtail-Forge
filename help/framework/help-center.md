The Help Center is the in-app product manual for Longtail Forge. Use it when you need the current behavior of the framework, active modules, search, files, tags, settings, or workspace-specific features without leaving the app.

Framework articles explain app-wide behavior that is always part of Longtail Forge, such as workspaces, permissions, clients and projects, search, files, tags, settings, notifications, and Help itself. Framework Help stays available across workspaces because it explains the shared app shell and shared services.

First-party module articles explain workflow areas that Longtail Forge ships with, such as Time Tracking, Tasks, Notes, Lists, and other built-in modules that can be enabled for a workspace. These articles appear only when their owning module is active and visible in the current workspace.

Third-party modules are external or separately maintained modules that can use the same contribution contracts when they are supported by the installed app. Their Help content should appear as module-owned product documentation, with source metadata that makes ownership clear, rather than pretending to be framework Help.

Article navigation follows the editable Help table of contents. Article cards and article detail views show source metadata so framework, first-party module, and third-party module ownership stay distinct where that matters.

Use Help for product guidance inside the app. Use the roadmap and repository documentation for planning, release history, implementation details, and engineering decisions.
