Longtail Forge is organized around work that can be found, started, paused, and resumed without rebuilding context from memory. The app links clients, projects, tasks, time entries, notes, lists, files, tags, notifications, search, and Help through the active workspace.

Use the Dashboard as the main summary screen after signing in. It gives a broad view of the active workspace without replacing the detailed module screens where the work actually happens. Use Workbench for live work, next actions, active timers, and resumable context.

Check the workspace switcher when records or modules look different than expected. Most records, settings, search results, and Help visibility are scoped to the active workspace, so switching workspaces can change what you see.

Clients and projects provide business context. Tasks hold commitments, outcomes, next actions, due dates, blockers, checklists, reminders, and task timers. Time Tracking records active and completed effort. Notes keep working context and reference material near the records they support. Lists handle operational checklists, reusable workflows, procurement-style work, and item progress. Files preserve source material and artifacts on supported records.

Search is the recovery surface when you do not remember where something lives. Use Search from the navigation row to find indexed records and Help articles across the active workspace. Tags help classify supported records for grouping and retrieval, but they do not replace permissions, status, workflow state, or billing logic.

Use notifications and reminders to recover attention when timing matters. Use Settings for user preferences, workspace settings, module controls, audit logs, and Help. When a workflow feels scattered, look for the linked record, next action, resume note, file attachment, tag, or search result that carries the work context forward.
