Notes are internal working records. A note can stand alone in the Notes Library or link to a client, project, task, ticket, or user when that context helps explain the work.

Each note has a title, Markdown body, Library bucket, status, visibility, owner, optional linked context, tags, files, and revision history. Notes follow the same module-state and permission checks as the rest of Longtail Forge, so a Library bucket, collection, tag, or linked record never grants access by itself.
